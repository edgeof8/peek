# Contributing to peek

Thank you for your interest in contributing to peek! This project is simple by design, but all contributions are welcome.

## How to Contribute

### Reporting Bugs
If you find a bug, please open an issue on GitHub with:
- A clear description of the problem
- Steps to reproduce it
- What you expected vs. what happened
- Your browser and OS

### Suggesting Features
Feature suggestions are welcome! Please open an issue with:
- A clear description of the feature
- Why it would be useful
- How it fits with peek's philosophy (lightweight, zero dependencies, one-click)

### Code Contributions
1. Fork the repository
2. Create a feature branch (`git checkout -b feature/amazing-feature`)
3. Make your changes
4. Test thoroughly
5. Commit your changes (`git commit -m 'Add amazing feature'`)
6. Push to the branch (`git push origin feature/amazing-feature`)
7. Open a Pull Request

## Code Style

Since peek is a bookmarklet, keep it:
- **Lightweight**: Minimize code size
- **Simple**: No complex abstractions
- **Clear**: Use readable variable names
- **Focused**: Solve one problem well

## Testing

When modifying the bookmarklet:
1. Test on multiple browsers (Chrome, Firefox, Safari, Edge)
2. Test on both desktop and mobile
3. Test on various websites (news sites, blogs, paywalled content)
4. Verify the anti-loop protection works
5. Check that URL encoding handles special characters correctly

## Questions?

Open an issue or reach out! peek is a community project, and we want to help.
